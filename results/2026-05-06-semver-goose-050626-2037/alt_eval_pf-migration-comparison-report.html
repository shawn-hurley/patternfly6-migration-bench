ISSUES: PageSection variant Mapping (TC062, TC066, TC067)

The issue

PF6 PageSection accepts only variant="default" or variant="secondary". The correct mapping is:

  • variant="light" → variant="default"
  • variant="dark" / variant="darker" → variant="secondary"

The migration branch maps all variants to "secondary", which is wrong for variant="light" (should be "default").

This is a semantic error — light/default sections will render with secondary (darker) styling.

ISSUES: CSS Variable Renames Not Completed (TC072, TC078)

The issue

CSS class prefixes were updated from pf-v5 to pf-v6 correctly. However, the actual CSS variable property names were not updated:
